# 여행 준비 보고서 프로그램 기획

## 📋 프로젝트 개요

### 주제
여행 준비를 위한 보고서 작성 프로그램

### 타겟 사용자
- **페르소나**: 김철수 (32세, 남성)
- **특징**: 절약 정신이 투철하고, 해외여행 경험이 없어 무엇을 준비해야 할지 막막한 초보 여행자
- **니즈**: 처음 일본 여행을 준비하며 체계적인 준비 가이드 필요

### 프로그램 작동 방식
1. **사용자 입력**: 여행할 지역 + 관심 키워드
2. **검색 Agent**: 웹 서치로 최신 정보 수집
3. **보고서 작성 Agent**: 템플릿에 맞춰 보고서 생성
4. **전달**: 이메일로 보고서 발송

## 🔍 기술 구현 시 핵심 포인트

### 1. 검색 Agent의 중요성

#### 2단계 검색 구조
```python
# Phase 1: 필수 법적 요구사항 검색 (최우선)
legal_queries = [
    f"{destination} 입국 요건 외교부",
    f"{destination} 비자 규정",
    f"{destination} 여행자 의무사항",
    f"{destination} 거주지 등록",
]

# Phase 2: 템플릿 기반 정보 수집
template_queries = [
    f"{destination} 항공권 저렴한 시기",
    f"{destination} 숙박 추천",
    f"{destination} 통신 USIM",
    f"{destination} 현지 교통",
    f"{destination} 필수 앱",
]
```

#### 신뢰도 높은 소스 우선
- ✅ 외교부, 대사관 공식 사이트
- ✅ 공식 관광청
- ✅ 정부 기관
- ⚠️ 개인 블로그 (최신 날짜 확인)
- ❌ 오래된 정보

### 2. 정보의 신선도 확보

#### 검색 쿼리 최적화
```python
search_queries = [
    # 법적 정보
    f"{destination} 비자 요건 2025",
    f"{destination} 입국 규정 최신",
    
    # 실시간 정보
    f"{destination} 항공권 가격",
    f"{destination} 환율 오늘",
    
    # 현지 정보
    f"{destination} 택시 앱 추천",
    f"{destination} 통신사 USIM 2025",
]
```

#### 크로스 체크
- 같은 정보를 3곳 이상에서 확인
- 날짜가 최근인 소스 우선

## 📝 우즈베키스탄 사례에서 배운 교훈

### 놓친 중요 정보: 거주지 등록

#### 규정
- 입국 후 근무일 기준 3일 이내 거주지 등록 필수
- 호텔에서 자동 등록 (영수증 보관)
- 위반 시 벌금 65만~150만원
- 출국 시 영수증 확인

#### 왜 놓쳤는가?
1. ❌ 템플릿에 명시되지 않음
2. ❌ 일반적인 패턴 위주로 생성 (대부분 국가는 이런 요구사항 없음)
3. ❌ 웹 검색을 하지 않음
4. ❌ 국가별 특수성 고려 안 함

#### 개선 방안
1. ✅ "필수 법적 요구사항" 섹션을 최우선으로
2. ✅ 외교부/대사관 정보를 가장 먼저 검색
3. ✅ 국가별 특수 규정 DB 구축
4. ✅ 검증 단계 추가

   - 잘못된 정보는 여행사에게 치명적
   - 출처와 최신성 명시 필수

5. **비즈니스 이해가 차별화 포인트**
   - 단순 개발자가 아닌 "문제 해결자"
